Pi.

The minimal agent harness that strudel is built on — traced from process start to provider call, and where an extension hooks in.

The philosophy

“A minimal agent harness. Adapt Pi to your workflows, not the other way around.”

Pi's stated design goal is extensibility as primitives, not bundled features — features other agents bake in, you build yourself. That is not a footnote. It is the entire reason strudel can exist as an extension rather than a fork.

— pi.dev · Mario Zechner / Earendil Works · MIT

The shape in one breath

Four packages. A pure core, an extensible shell.

A strictly layered TypeScript monorepo. The loop at the bottom is a pure, side-effect-free event emitter — it takes a context snapshot, streams a response, runs tools, emits typed events. Everything that touches the world — persistence, extensions, compaction, UI — lives in the shell that subscribes to those events.

core (agent-core)

The loop. Stateless. No this, no I/O. Emits the events everything else listens to.

ai

One call boundary. streamSimple() — every provider behind a single streaming protocol.

coding-agent

The shell. Session, extensions, compaction, skills, pi install, TUI wiring. The seam lives here.

tui

Replaceable. Terminal-UI primitives. The loop never imports it.

One turn, start to end

From a keystroke to the model and back.

The kernel

The loop is pure.
The shell is not.

This single boundary explains the whole codebase. agent-loop.ts has no class, no state, no file access — it receives a snapshot and emits events. Persistence, UI, and every extension hook live in the subscribers. That is why a turn is testable with a fake stream, why the UI is swappable across interactive / print / RPC unchanged, and why an extension can reshape a turn without ever touching the engine.

why it matters for strudel

Strudel never modifies the loop. It lives entirely in the subscriber layer — which is exactly why it can be a clean extension, and why it physically cannot sprawl into the runtime the way the lab did.

How the packages compose

Dependencies point one way: down.

coding-agent depends on all three; agent-core depends on ai; tui and ai stand alone. An extension author uses coding-agent (the seam) and takes types only from the rest.

The extension seam · ★ = can mutate the turn

~25 hooks. A handful change everything.

input★ transform or swallow the user's text before anything sees it
before_agent_start★ rewrite the system prompt · inject context messages — strudel's main hook
context★ rewrite the message array before each LLM call (RAG, distillation)
tool_call★ patch arguments or block a tool outright
tool_result★ transform a result before the model sees it
message_end★ post-process finalized output
turn_* / message_* / tool_execution_*observe only — the telemetry surface

Plus capabilities: registerTool, setActiveTools (lock the surface), registerProvider, exec. Extensions load via jiti from .pi/extensions/; pi install npm:… / git:… distributes them.

What Pi already does — and where it breaks

Skills are dumped into the prompt. All of them.

Pi discovers every skill and injects them into the system prompt as <available_skills> XML — name, description, location — every turn. The model is told to read the file when relevant. Skills are text-injection, not execution.

this is the thesis, confirmed in the source

That dump-everything mechanism is exactly what breaks at scale — 100 skills in the prompt, the agent reaches for the same five. Strudel doesn't rebuild skills; it replaces the selection mechanism. Also confirmed: tools merge via registerTool; slash-commands are code or text macros; no native MCP in 0.74 — that's open territory.

What Mario optimized for

The principles, read off the architecture.

pure loop, dirty shell

Side effects live only in event subscribers. The engine stays a function.

extensions first-class

Every observable moment is a typed hook. Extensions can even provide resources.

one provider boundary

streamSimple() is the sole LLM call site. Wire format never leaks into the loop.

the JSONL is the truth

Each message streamed to disk on finish. Survives a mid-turn crash; rebuilt on open.

stale-aware context

Extension handles throw after reload instead of dangling. Bugs die loudly.

UI is replaceable noise

One session class across interactive / print / RPC. The UI is wiring at the top.
